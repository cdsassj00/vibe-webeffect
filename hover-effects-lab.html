브라우저의 재발견 · 마우스 호버

마우스를 올리면 뭔가 일어난다 — 그 효과의 이름

코드를 외우려는 게 아니다. 이 효과의 이름을 눈으로 찾아 AI에게 “이거 리프트로 해줘”처럼 정확히 지시하기 위한 용어집이다. 카드의 ‘?’로 준비물·지시문·코드를 확인하라.

↻ 모든 데모는 자동으로 반복 재생됩니다. 마우스를 올리면 직접 조작도 가능해요.

01리프트lift · elevate
transform: translateY(−24px)

리프트 (엘리베이션)

필요한 것

순수 CSS. 요소를 위로 올리고 그림자를 키우면 ‘떠오르는’ 느낌. 카드·버튼에 가장 흔히 쓴다.

AI에게 이렇게

“호버하면 카드가 위로 떠오르며 그림자가 커지게 해줘.”

핵심 코드
.card:hover{ transform: translateY(-24px); box-shadow: 0 34px 54px -16px #000; }
023D 틸트3d tilt
3D
perspective() rotateX/Y()

3D 틸트

필요한 것

커서 위치를 rotateX/rotateY로 변환. 자동 데모는 계속 기울고, 마우스를 올리면 커서를 따라 기운다. perspective가 3D의 핵심.

AI에게 이렇게

“카드가 마우스 위치에 따라 3D로 기울어지게 해줘(perspective + rotateX/Y).”

핵심 코드
el.style.transform = `perspective(600px) rotateX(${rx}deg) rotateY(${ry}deg)`;
03글로우glow · bloom
box-shadow: 0 0 46px 컬러

글로우 (블룸)

필요한 것

순수 CSS. 색이 번지는 box-shadow + 살짝 밝기(brightness). CTA 버튼 강조에 좋다.

AI에게 이렇게

“호버하면 버튼 주변으로 색이 은은하게 빛나게(글로우) 해줘.”

핵심 코드
.btn:hover{ box-shadow: 0 0 46px #54c7ff; filter: brightness(1.18); }
04샤인 스윕shine · sheen
::after 광택 바 이동

샤인 스윕

필요한 것

순수 CSS. 기울인 흰 그라디언트 바(::after)를 요소 위로 지나가게 한다. overflow:hidden 필수.

AI에게 이렇게

“호버하면 버튼 위로 빛이 한 번 스윽 지나가는 샤인 효과 넣어줘.”

핵심 코드
.btn::after{ /* 흰 그라디언트 바 */ left:-90%; } .btn:hover::after{ left:150%; transition:.6s; }
05언더라인 리빌underline reveal
텍스트 링크
background-size: 0% → 100% 3px

언더라인 리빌

필요한 것

순수 CSS. 밑줄을 background 그라디언트로 만들고 background-size의 너비를 0→100%로 늘린다. 링크·메뉴에 필수.

AI에게 이렇게

“링크에 호버하면 밑줄이 왼쪽에서 오른쪽으로 그어지게 해줘.”

핵심 코드
a{ background:linear-gradient(#54c7ff,#54c7ff) 0 100%/0% 3px no-repeat; transition:.35s; } a:hover{ background-size:100% 3px; }
06이미지 줌image zoom
ZOOM
overflow:hidden + scale(1.26)

이미지 줌 (Ken Burns)

필요한 것

순수 CSS. 이미지 컨테이너에 overflow:hidden, 안의 이미지를 scale로 확대. 프레임은 고정, 이미지만 커진다.

AI에게 이렇게

“썸네일에 호버하면 안의 이미지만 천천히 확대되게 해줘(프레임은 고정).”

핵심 코드
.thumb{ overflow:hidden; } .thumb:hover img{ transform:scale(1.26); }
07스포트라이트cursor glow
커서를 따라온다
radial-gradient(at var(--mx) var(--my))

스포트라이트 (커서 글로우)

필요한 것

마우스 좌표를 CSS 변수(--mx, --my)로 넘기고, 그 지점을 중심으로 radial-gradient 빛을 그린다. SaaS 카드에서 자주 본다.

AI에게 이렇게

“카드 위에서 마우스를 따라 빛이 움직이는 스포트라이트 효과 만들어줘.”

핵심 코드
el.style.setProperty('--mx', x+'px'); /* CSS: radial-gradient(circle at var(--mx) var(--my), #fff3, transparent) */
08마그네틱magnetic pull
translate(dx×0.4, dy×0.4)

마그네틱 (자석 끌림)

필요한 것

버튼이 커서 쪽으로 살짝 끌려간다. 커서와의 거리(dx,dy)에 배율을 곱해 translate. 자동 데모는 보이지 않는 점을 따라 움직인다.

AI에게 이렇게

“버튼이 근처 마우스 쪽으로 자석처럼 살짝 끌려오게 해줘.”

핵심 코드
el.style.transform = `translate(${dx*0.4}px, ${dy*0.4}px)`; // 벗어나면 translate(0,0)로 복귀
09컬러 필fill swipe
::before scaleX(0) → scaleX(1)

컬러 필 (스와이프)

필요한 것

순수 CSS. 배경 블록(::before)을 scaleX(0→1)로 채우고 글자색을 뒤집는다. 아웃라인 버튼에 자주 쓴다.

AI에게 이렇게

“아웃라인 버튼에 호버하면 색이 왼쪽에서 차오르고 글자색이 반전되게 해줘.”

핵심 코드
.btn::before{ transform:scaleX(0); transform-origin:left; } .btn:hover::before{ transform:scaleX(1); }

9가지 호버 효과 — 대부분 순수 CSS :hover. ②⑦⑧(틸트·스포트라이트·마그네틱)만 마우스 좌표를 JS로 넘긴다. 각 카드 ‘?’에 준비물·AI 지시문·코드.
