브라우저의 재발견 · 등장·리빌

화면에 들어올 때 비로소 나타난다

스크롤해서 요소가 화면에 진입하는 순간 재생되는 등장 효과 9가지. 실전에서는 IntersectionObserver(요소가 화면에 보이는지 감시하는 브라우저 내장 기능) 하나로 전부 구동한다. 카드의 ‘?’에 준비물·AI 지시문·핵심 코드.

↻ 데모는 자동 반복 재생 — 실제로는 화면 진입 시 1회 재생된다.

01페이드 업fade up
CONTENT
opacity 0→1 + translateY 34→0

페이드 업 (기본 중의 기본)

필요한 것

숨김 상태(투명+아래로 이동)를 CSS로 정의하고, 화면 진입 시 클래스 하나를 붙인다. 모든 등장 효과의 뼈대.

AI에게 이렇게

“섹션이 화면에 들어오면 아래에서 위로 떠오르며 나타나게 해줘. IntersectionObserver로.”

핵심 코드
new IntersectionObserver(es=>es.forEach(e=>{ if(e.isIntersecting) e.target.classList.add("show"); }),{threshold:0.2}).observe(el);
02스태거 그리드stagger children
transition-delay = i × 80ms

스태거 그리드 (자식 순차 등장)

필요한 것

카드 목록·갤러리에서 자식들이 도미노처럼 순서대로 등장. 각 자식에 transition-delay: i×80ms만 주면 끝.

AI에게 이렇게

“카드 그리드가 화면에 들어오면 카드들이 순서대로 하나씩 나타나게(스태거) 해줘.”

핵심 코드
items.forEach((el,i)=> el.style.transitionDelay = i*80+"ms");
03클립 와이프clip reveal
REVEAL
clip-path inset(0 100% 0 0) → 0

클립 와이프 리빌

필요한 것

요소를 clip-path로 완전히 잘라뒀다가 진입 시 잘린 영역을 0으로. 이미지·배너가 닦이듯 드러난다. 방향은 inset의 변만 바꾸면 된다.

AI에게 이렇게

“이미지가 화면에 들어오면 왼쪽부터 닦이듯이 드러나는 클립 리빌 넣어줘.”

핵심 코드
el{clip-path:inset(0 100% 0 0);transition:clip-path .8s} el.show{clip-path:inset(0 0 0 0)}
04라인 마스크 리빌line mask
텍스트가 줄 단위로 아래에서 밀고 올라온다
overflow:hidden + translateY(115%)

라인 마스크 리빌 (고급 타이포)

필요한 것

각 줄을 overflow:hidden 상자로 감싸고 안쪽 텍스트를 아래(115%)에 숨겨뒀다 줄마다 시차를 두고 올린다. 어워즈 사이트 헤드라인의 표준.

AI에게 이렇게

“헤드라인이 줄 단위로 마스크 아래에서 밀고 올라오는 라인 리빌 효과 만들어줘.”

핵심 코드
.ln{overflow:hidden} .ln span{transform:translateY(115%)} .show .ln span{transform:none} /* +줄별 delay */
05스케일 + 블러 인scale · blur in
FOCUS
scale(.6) + blur(10px) → 원상

스케일 + 블러 인

필요한 것

작고 흐릿한 상태에서 커지며 또렷해지는 등장. 숫자·키워드 강조에 잘 어울린다. filter는 GPU 부담이 있으니 큰 영역엔 남용 금지.

AI에게 이렇게

“키워드가 흐릿하고 작게 시작해서 커지며 선명해지는 등장 효과 넣어줘.”

핵심 코드
el{opacity:0;transform:scale(.6);filter:blur(10px)} el.show{opacity:1;transform:none;filter:blur(0)}
06좌우 교차 슬라이드alternate slide
왼쪽에서오른쪽에서다시 왼쪽에서
translateX(±56px) 홀짝 교차

좌우 교차 슬라이드

필요한 것

목록의 홀수는 왼쪽, 짝수는 오른쪽에서 미끄러져 들어온다. :nth-child(odd/even)로 방향만 갈라주면 끝. 타임라인·피처 목록에.

AI에게 이렇게

“항목들이 홀수는 왼쪽, 짝수는 오른쪽에서 교차로 슬라이드되며 나타나게 해줘.”

핵심 코드
li:nth-child(odd){transform:translateX(-56px)} li:nth-child(even){transform:translateX(56px)} .show li{transform:none;opacity:1}
07카운트업 온뷰count-up
0ACTIVE USERS
ease-out 보간으로 0 → 목표값

카운트업 (보일 때 세기)

필요한 것

통계 숫자가 화면에 보이는 순간 0부터 목표값까지 가속-감속으로 센다. 진입 전에 세면 아무도 못 보니 IntersectionObserver가 필수인 대표 사례.

AI에게 이렇게

“통계 숫자가 화면에 보이면 0부터 목표값까지 차오르는 카운트업 애니메이션 넣어줘.”

핵심 코드
const t=Math.min(1,(now-t0)/1200); el.textContent=Math.round(target*(1-Math.pow(1-t,3))) .toLocaleString();
08커튼 이미지 리빌curtain + zoom
IMAGE
overlay scaleX 1→0 + img 1.25→1

커튼 이미지 리빌

필요한 것

이미지 위 색 커튼이 옆으로 걷히는 동시에 이미지가 살짝 줌아웃(1.25→1). 두 모션의 합이 에디토리얼한 고급감을 만든다.

AI에게 이렇게

“이미지 위 색 오버레이가 옆으로 걷히면서 이미지가 살짝 줌아웃되는 커튼 리빌 만들어줘.”

핵심 코드
.ov{transform-origin:right} .show .ov{transform:scaleX(0)} .show img{transform:scale(1)} /* 1.25에서 */
09타임라인 드로잉line draw
width 0 → 130px (선이 자란다)

타임라인 라인 드로잉

필요한 것

점이 찍히고 선이 자라 다음 점에 닿는다. 연혁·프로세스 다이어그램에서 진입할 때마다 구간이 이어지는 연출. SVG면 stroke-dashoffset으로도 동일.

AI에게 이렇게

“타임라인이 화면에 들어오면 점이 찍히고 선이 자라나며 이어지게 해줘.”

핵심 코드
.seg{width:0;transition:width .9s} .show .seg{width:130px} /* SVG: stroke-dashoffset L→0 */

9가지 등장 효과 — 전부 같은 골격이다: CSS로 숨김 상태 → 표시 상태를 정의하고, IntersectionObserver가 화면 진입 순간 클래스를 붙인다. 관찰 1회 후 unobserve()하면 다시 안 나온다(1회성), 안 하면 스크롤마다 재생된다.
